Everythin-Tag Project

Background

Please search 钥匙寻找器( key-founder ) in taobao.com, you will find numberless result items for sale.
However they are for local use, if your key-founder is out of the range of your detector( such as your iPhone ), you will probably get it lost forever.

On the other side of the planet, Tile is growing quickly.
Tile, a lost-and-found product, which centers the position datas into its server and then provides position info to the device-owner.
The difference of tile is that by users helping each other to search for lost items,
items seems less possible to get lost even it is out of it’s owner’s scope.

Goals

But tile only would like to share info/data for it’s own users.
Which greatly limits it’s potential ability. Imagine, what if we combine the local-key-finders producers and tile-like platform together?

Demo

The 0.2 demo is available on android play when you try to search everything-tag client on it.

Intros

The goal looks nice, but we need to do it from scratch.
The best way is to download it from Android Studio.
By tolerating the crashes, you will get a direct impressing of it more quickly.

Local search( saved items )

Users are not required to register, if they only want do to the local search.
By local search, they can locate the target’s distance by Bluetooth signal strength.

Global Search

By registration, users will have an account which could access the datas on the server.
When one item is marked as lost, the server will push it’s latest info to the cellphone to help the owner.

Fidelis Learning Relationship Management (LRM)

Abstract

According to the health report published by Centers for Disease Control and Prevention (CDC 2009),
nearly one-third of people over age of 65 suffer from severe falling injuries.

Our “Fallarm” project is to develop a simulating remote monitoring system would be a solution
to watch for patient activity and send alert message to caregivers directly on their device.

System Structure

This whole system includes: a web server which handles browser requests,
a MySQL database which stores all th patients' and doctors' data, an android app which monitors patients,
a network server which handles request or react with android app.

System Design

System designing models includes: UML framework, Use case diagrams, class diagrams, sequence diagrams.

User Interface

Using a wordpress template, the system ui looks better and developed faster.

User Interface

Selenium is used to provide auto testing. and also google app engine is used to provide map service.
